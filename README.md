📌 Added Feature: Dynamic Emoji Based Sentiment System for Tickets

Implemented an intelligent dynamic sentiment analysis system for customer support tickets, enabling real-time detection of customer emotions such as anger, frustration, sadness, or satisfaction based on message content.

The system allows admins to define custom keywords mapped to emojis, making sentiment detection fully configurable without code changes.


🚀 Key Features

* Keyword-Based Sentiment Detection
  Automatically analyzes ticket messages and identifies sentiment based on predefined keywords.

* Dynamic Emoji Mapping
  Admin can assign custom emojis to specific keywords (e.g., angry 😡, happy 😊, frustrated 😤).

* Real-Time Sentiment Display
  Customer emotion is displayed dynamically on tickets using emojis.

* Average Sentiment Analysis
  System calculates overall customer sentiment trends across tickets.

* Admin Configurable Rules
  No hardcoding required — admins can add/update/remove sentiment keywords anytime.

* Improved Support Insights
  Helps support teams quickly understand customer mood and prioritize responses.



🎯 Objective

To enhance customer support efficiency by introducing a flexible, keyword-driven sentiment system that visually represents customer emotions and improves response prioritization.
